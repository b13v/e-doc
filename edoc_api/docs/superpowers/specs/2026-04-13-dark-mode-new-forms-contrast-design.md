# Dark-Mode Contrast Fix For New Invoice/Act Forms

## Context
- Affected pages: `/invoices/new` and `/acts/new`.
- Current issue in dark mode:
  - Top mode radio area (`From contract` / `Direct without contract`) has low contrast text over muted background.
  - Bottom items block appears with light-surface styling and poor contrast.
  - Dynamically added item rows must keep the same contrast as initial visible UI.

## Goal
Make both pages clearly readable in dark mode without changing business logic, page structure, or localization behavior.

## Scope
- In scope:
  - Tailwind class fixes in:
    - `lib/edoc_api_web/controllers/invoice_html/new.html.heex`
    - `lib/edoc_api_web/controllers/act_html/new.html.heex`
  - Dynamic row markup classes generated by JavaScript in:
    - `invoice_html/new.html.heex` `addItemRow()` template string
    - `act_html/new.html.heex` `addItemRow()` template string
  - Regression tests for dark-mode class contracts.
- Out of scope:
  - Redesign of forms/layout.
  - Behavior changes in contract/direct modes.
  - Changes outside these two pages and related tests.

## Chosen Approach
### Option A (selected): Template + JS class parity fix
- Add missing `dark:*` classes for:
  - mode-toggle card shell
  - toggle label text
  - items section heading
  - items surface shell
  - dynamic row labels/inputs/selects/rings
- Reason: minimal, low-risk fix aligned with existing Tailwind style patterns.

### Option B (not selected): extract shared form-surface component
- Cleaner long-term abstraction but larger blast radius for a bugfix.

## Design Details
### 1) Mode Toggle Contrast
- Update both pages’ mode-toggle container from light-only classes to dual light/dark classes.
- Ensure labels use `dark:text-slate-100`.

### 2) Items Block Surface Contrast
- Update items section heading with `dark:text-slate-100`.
- Update items surface wrapper with `dark:bg-slate-900/80`.
- Keep spacing/border/radius unchanged.

### 3) Dynamic Item Rows
- For JS-generated rows in `addItemRow()` on both pages, ensure:
  - labels include `dark:text-slate-300`
  - input/select fields include `dark:bg-slate-800` and `dark:text-slate-100`
  - ring classes include `dark:ring-slate-600`
- Preserve existing field names and JS behavior.

## Testing Strategy
### Regression test (first)
- Add or extend `test/edoc_api_web/controllers/workspace_overview_ui_test.exs`:
  - Render direct mode pages:
    - `/invoices/new?invoice_type=direct`
    - `/acts/new?act_type=direct`
  - Render contract mode pages:
    - `/invoices/new?invoice_type=contract`
    - `/acts/new?act_type=contract`
  - Assert presence of expected dark-mode class contracts for:
    - mode-toggle card shell
    - mode label contrast
    - items block shell
    - items heading contrast
    - dynamic row class tokens in rendered `addItemRow()` JS template strings
  - Test intent: server-rendered class-contract assertions only (no browser runtime execution in this test suite).

### Verification
- Run targeted tests:
  - new regression test location
  - existing invoice/act new form-chrome tests in same test module
- Run full `workspace_overview_ui_test.exs` file to ensure no regressions in shared workspace shell contracts.

## Error Handling / Risk
- Risk: class-token assertions can be brittle if class order changes.
- Mitigation: assert stable token substrings and semantic class groups instead of full exact class strings where possible.

## Success Criteria
- In dark mode, mode-toggle text is clearly readable on both pages.
- Items block is visually consistent with dark theme.
- Dynamically added rows remain readable.
- Regression tests pass and existing workspace UI tests stay green.
